Проект реализует игру тетрис для компьютера на C# с WinForms. Реализованы следующие функции:
- Генерация фигур тетрамино
- Вращение фигур
- Перемещение фигур
- Размещение фигур на игровом поле
- Удаление заполненных рядов игрового поля
- Окончание игры
- Подсчет количества очков

Проблема: необходимость создавать фигуры, имеющие разные реализации алгоритма вращения и перемещения.
Решение: Фабричный метод.

Паттерн позволяет создавать фигуры, имеющие разные алгоритмы вращения Rotate() и перемещения Move(), не отвлекаясь на детали реализации в основной программе.

Программа содержит:
Абстрактный продукт Tetromino (общий интерфейс для всех видов фигур)
Конкретные продукты: IPiece, OPiece, TPiece …
Интерфейс IPieceProvider, объявляющий фабричный метод Create()
Конкретные фабрики IProvider, OProvider, TProvider …
Клиент Game, реализующий игровую логику
Класс Board, отвечающий за хранение и манипуляции с игровой доской
Класс Form1, отрисовывающий доску и фигуры, а также регистрирующий нажатия клавиш.
Добавление новой фигуры сводится к добавлению нового класса фигуры и новой фабрики. Без необходимости изменения классов Game, Form1.
Board возможно будет нужно обрабатывать разные классы по-разному, так что он знает про некоторые виды фигур.

После внедрения паттерна:
Упростилось расширение функционала
Установилась слабая связность (Game не зависит от конкретных классов фигур)
Инкапсуляция создания фигур (целиком в конкретной фабрике)
