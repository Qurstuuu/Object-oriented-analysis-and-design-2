# Клиент для запроса погоды.
* Проект реализует десктоп приложение для получения прогноза погоды в разных городах на C++ с QT.
### Реализованы следующие функции:
* Получение координат по введённому адресу (по API Яндекс.Геокодер)
* Получение данных о погоде по координатам (по API Open-Meteo)
* Отображение температуры, состояния погоды, влажности и ветра
* Кэширование результатов для быстрого доступа (двухуровневое: память + диск)
* Асинхронная работа с сетью (неблокирующий интерфейс)
* Обработка ошибок сети и таймаутов

### Проблема:
Взаимодействие с двумя разными API, управление асинхронными запросами, кэширование, парсинг JSON, обработка ошибок делаются на форме (о бедный UI-программист).
## Решение: Паттерн <b>Фасад</b>

### Программа содержит:
1. Форма MainWindow - отвечает за графический интерфейс и взаимодействие с пользователем
2. Фасад WeatherFacade - реализует всю логику взимодействия с API Яндекс.Геокодера и Open-Meteo, обрабатывает запросы, кэширует ответы, отслеживает таймаут.
3. Структура WeatherData - хранит информацию о погоде: название адреса, температура, состояние, влажность, ветер, время обновления.

#### Упростилось расширение функционала
* Для добавления нового источника данных (например, OpenWeatherMap вместо Open-Meteo) достаточно изменить только фасад, не трогая MainWindow
* Для добавления нового типа кэширования (например, база данных) изменения локализуются внутри фасада

#### Установилась слабая связность
* MainWindow не знает о существовании QNetworkAccessManager, QJsonDocument, QSettings
* MainWindow не зависит от конкретных API (Яндекс.Геокодер, Open-Meteo)
* Замена API или добавление нового источника данных не требует изменений в интерфейсе

#### Инкапсуляция сложности
* Вся работа с сетью скрыта внутри фасада
* Парсинг JSON (для двух разных API) инкапсулирован в приватных методах
* Кэширование (память + диск) полностью скрыто от клиента

#### Улучшилась тестируемость
* Фасад можно тестировать отдельно, без запуска графического интерфейса
* Можно подменить реальные сетевые запросы на тестовые данные
* Управление асинхронными запросами и таймаутами — ответственность фасада
